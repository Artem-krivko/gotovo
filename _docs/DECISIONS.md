---

## 2026-04-19

### ✅ Не обучаем свою модель
**Решение:** Используем Claude API для генерации HTML/CSS дизайна
**Альтернативы:** Обучить собственную модель (как Lovable/v0)
**Причина:** Обучение требует миллионы долларов и команду ML. Claude API даёт тот же результат за копейки. Под капотом Lovable и v0 используют те же LLM.

### ✅ Без базы данных на старте (Фаза 1-2)
**Решение:** Заявки сохраняются в JSON + email нотификация
**Альтернативы:** PostgreSQL (Supabase), MongoDB Atlas
**Причина:** YAGNI. На старте до 50 заявок/месяц JSON хватит. БД добавим когда появится реальная нагрузка.

### ✅ Без авторизации на старте
**Решение:** Нет личного кабинета, нет логина
**Альтернативы:** NextAuth с Google OAuth
**Причина:** Лишняя сложность для MVP. Цель — получить первые заявки, не строить SaaS.

### ✅ iframe для превью генератора
**Решение:** Рендер HTML от Claude в `<iframe srcDoc={...}>`
**Альтернативы:** Парсить HTML и рендерить через React, открывать в новой вкладке
**Причина:** Полная изоляция, безопасность (sandboxed iframe), не нужно парсить HTML. Открытие в новой вкладке — плохой UX.

### ✅ Vercel для деплоя
**Решение:** Vercel Hobby (бесплатно)
**Альтернативы:** Railway, Render, VPS
**Причина:** Zero-config для Next.js, автодеплой из GitHub, бесплатные serverless функции, встроенная аналитика.

### ✅ Resend для email
**Решение:** Resend.com API
**Альтернативы:** SendGrid, Nodemailer + SMTP, EmailJS
**Причина:** Современный, бесплатный tier (3000 писем/месяц), простой SDK для TypeScript.
