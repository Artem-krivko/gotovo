## ЧТО УЖЕ ПРОБОВАЛИ (и почему не зашло)

1. **Particle system из PNG-пикселей** — canvas с 1800 частицами которые собирались в форму UI. Результат: непонятные мерцающие точки. Пользователь не понимал что происходит.

2. **HTML-блоки которые вылетают** — DOM-карточки (навбар, метрика +248%, браузерный мокап, форма) летели из разных сторон через GSAP. Результат: выглядело дёшево — заглушки вместо реального UI.

3. **Scroll storytelling с 5 шагами** — sticky секция 500vh где при скролле менялся текст шагов + правая анимация. Проблема: анимация не впечатляла, текст читался нормально, но правая панель была слабой.

**Главный вывод:** Анимация ради анимации не работает. Нужен либо реальный визуал (скриншот, видео, живой iframe) либо очень качественная 3D-сцена.

## ЗАДАЧА ДЛЯ НОВОГО ЧАТА

### Сначала — стратегия (не кодируй сразу!)

Предложи **3 конкретных концепции** улучшения главной страницы. Для каждой:

1. **Название концепции**
2. **Что именно изменится** (hero? все секции? конкретный элемент?)
3. **Технология** (GSAP, Three.js, CSS 3D, WebGL, iframe, видео и т.д.)
4. **Сложность реализации** (1-5)
5. **Риск что выглядит дёшево** (1-5, где 5 = высокий риск)
6. **Ожидаемый wow-эффект** (1-5)
7. **Влияние на конверсию** (позитивное/нейтральное/негативное + почему)

### Критерии хорошей концепции
- Не требует внешних 3D-моделей или тяжёлых ассетов
- Работает на мобилке (хотя бы деградированная версия)
- Не роняет LCP ниже 2.5 сек
- Выглядит как топ-агентство, а не как "попробовали GSAP"
- Реализуемо за 1 сессию кодинга (3-4 часа)

### После обсуждения
Когда выберем концепцию — читай скилы из _ai-skills/ и реализуй пошагово.
