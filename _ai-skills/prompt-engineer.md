# SKILL: AI Промпт-инженер (Генератор дизайна)

> Читай этот файл при работе над промптами для генератора дизайна.
> Это один из самых важных файлов проекта — качество генерации зависит от него.

---

## Принцип работы генератора

Клиент вводит простое описание → код автоматически собирает мастер-промпт →
Claude API возвращает готовый HTML → рендерится в iframe клиенту.

Клиент не видит промпты. Клиент видит только красивый результат.

---

## Архитектура промпта (lib/prompts.ts)

Промпт состоит из трёх частей:

### Часть 1 — SYSTEM PROMPT (константа, никогда не меняется)
Это "личность" генератора. Пишется один раз.

```
Ты — экспертный веб-дизайнер и фронтенд-разработчик.
Твоя задача: создавать красивые, современные одностраничные сайты в виде
единого HTML файла.

ТЕХНИЧЕСКИЙ СТЕК (всегда используй):
- Tailwind CSS через CDN: <script src="https://cdn.tailwindcss.com"></script>
- Google Fonts (Inter или Manrope): через <link> тег
- Alpine.js для интерактивности (меню, аккордеон): через CDN если нужно
- Lucide Icons через CDN для иконок

ПРАВИЛА ДИЗАЙНА:
- Современный, профессиональный вид
- Тёмная или светлая тема в зависимости от бизнеса
- Реалистичный контент (не Lorem ipsum — придумай реальные тексты)
- Адаптивный дизайн (mobile-first)
- Плавные hover-эффекты на кнопках и карточках
- Чёткая иерархия: hero → секции → CTA → footer

СТРУКТУРА ВЫВОДА:
Возвращай ТОЛЬКО валидный HTML код.
Никаких объяснений, никакого markdown, никаких ```html блоков.
Только чистый HTML начиная с <!DOCTYPE html>
```

### Часть 2 — FEW-SHOT ПРИМЕРЫ (3-5 примеров хорошего результата)
Это примеры которые показывают Claude "вот как должно выглядеть".
Добавляются в system prompt как примеры пар input → output.
(Добавим после того как сгенерируем первые хорошие результаты вручную)

### Часть 3 — USER PROMPT (динамическая, собирается из формы клиента)
```
Создай сайт со следующими параметрами:

Бизнес: ${businessType}
Название: ${businessName || 'придумай подходящее'}
Описание от клиента: ${userDescription}
Стиль: ${style}
Язык контента: ${language}
Цветовые предпочтения: ${colorPreference || 'выбери сам исходя из типа бизнеса'}
Разделы сайта: ${sections.join(', ')}

Сделай сайт реалистичным: придумай убедительные тексты, цены, преимущества —
всё что подходит для этого типа бизнеса.
```

## Параметры формы клиента

```typescript
interface GeneratorParams {
  businessType: string      // 'стоматология' | 'кафе' | 'агентство' | ...
  businessName?: string     // необязательно
  userDescription: string   // свободный текст от клиента (ГЛАВНОЕ ПОЛЕ)
  style: 'modern' | 'minimal' | 'bold' | 'corporate' | 'creative'
  language: 'ru' | 'en' | 'de'
  colorPreference?: string  // 'синий и белый' | 'тёмная тема' | ...
  sections: string[]        // ['hero', 'about', 'services', 'pricing', 'contacts']
}
```

## UI формы для клиента (упрощённый)

Клиент НЕ заполняет все поля. Минимум:
1. **Большой textarea:** "Расскажите о своём бизнесе" (обязательно)
2. **Тип бизнеса:** dropdown с популярными вариантами (обязательно)
3. **Стиль:** визуальный выбор из 4 плиток с превью (необязательно, default: modern)
4. **Язык:** ru/en/de (необязательно, default: ru)

Всё остальное Claude решает сам на основе типа бизнеса.

## Библиотеки подключаемые через CDN (в HTML который генерирует Claude)

Все подключаются через `<script>` и `<link>` теги прямо в HTML — клиенту ничего устанавливать не нужно, всё работает в iframe:

| Библиотека | CDN URL | Зачем |
|-----------|---------|-------|
| Tailwind CSS | cdn.tailwindcss.com | Все стили |
| Alpine.js | cdn.jsdelivr.net/npm/alpinejs | Простая интерактивность |
| Google Fonts | fonts.googleapis.com | Красивые шрифты |
| Lucide Icons | unpkg.com/lucide@latest | Иконки |
| AOS (опционально) | unpkg.com/aos | Анимации при скролле |

## Фича: загрузка референс-скриншота (Фаза 2)

Claude умеет принимать изображения. Флоу:
1. Клиент загружает скриншот сайта который ему нравится
2. Фронтенд конвертирует в base64
3. Передаётся в API вместе с текстом:
```typescript
messages: [{
  role: 'user',
  content: [
    { type: 'image', source: { type: 'base64', media_type: 'image/jpeg', data: base64 }},
    { type: 'text', text: `Создай сайт в похожем стиле. Параметры: ${params}` }
  ]
}]
```
4. Claude видит стиль и воспроизводит похожий

## Итерация промптов (как улучшать)

1. Генерируй тестовые результаты для 5-10 разных типов бизнеса
2. Лучшие результаты сохраняй в `_docs/prompt-examples/` как few-shot примеры
3. Плохие результаты — анализируй что пошло не так и добавляй правило в system prompt
4. Повторяй до стабильно хорошего качества

**Метрика качества:** покажи результат 3 незнакомым людям. Если хотя бы 2 из 3 говорят
"вау, хорошо выглядит" — промпт хороший.
